Snelle Start

Welkom bij de Euramax intern beveiligingssysteem documentatie. Dit platform biedt AI-aangedreven phishing bescherming speciaal ontwikkeld voor Euramax medewerkers.

Let op: Dit is een intern systeem exclusief voor Euramax medewerkers. Toegang vereist interne authenticatie.

Systeemvereisten

  • Windows 10/11 of macOS 10.14+
  • Moderne webbrowser (Chrome 90+, Firefox 88+, Safari 14+)
  • Actieve VPN-verbinding met Euramax netwerk
  • Geldige Euramax Active Directory account

Eerste Toegang

1. Verbind met Euramax VPN
2. Navigeer naar https://security.internal.euramax.nl
3. Log in met je AD-credentials
4. Voltooi de initiÃ«le beveiligingstraining
5. Configureer je notificatie-instellingen

Installatie & Setup

Server Installatie

Voor IT-beheerders die het systeem installeren op nieuwe servers:

# Clone repository (intern GitLab)
git clone https://gitlab.internal.euramax.nl/security/phishing-protection.git

# Installeer dependencies
npm install

# Configureer environment
cp .env.example .env.production
nano .env.production

# Start productie server
npm run start:production

Database Setup

# Connectie met interne SQL Server
Server: sql-prod.internal.euramax.nl
Database: EuramaxSecurity
Authentication: Integrated (AD)

API Overzicht

Het Euramax beveiligingssysteem biedt een RESTful API voor integratie met interne systemen.

Base URL

https://security-api.internal.euramax.nl/v2/

Authenticatie

Alle API-calls vereisen een geldige JWT token verkregen via AD-authenticatie:

POST /auth/login
Content-Type: application/json

{
  "username": "gebruiker@euramax.nl",
  "password": "wachtwoord",
  "domain": "EURAMAX"
}

Response Format

{
  "success": true,
  "data": {...},
  "timestamp": "2024-03-15T10:30:00Z",
  "version": "2.1.4"
}

Bedreigingen API

Recente Bedreigingen Ophalen

GET /threats/recent
Authorization: Bearer {jwt_token}

Parameters:
- limit: aantal resultaten (max 100)
- department: filter op afdeling
- severity: filter op ernst (low, medium, high, critical)

Bedreiging Rapporteren

POST /threats/report
Content-Type: application/json
Authorization: Bearer {jwt_token}

{
  "type": "phishing_email",
  "source": "external_email",
  "target_email": "medewerker@euramax.nl",
  "subject": "Verdachte e-mail onderwerp",
  "sender": "verdacht@example.com",
  "severity": "high"
}

Development Setup

Lokale Ontwikkelomgeving

# Clone development branch
git clone -b development https://gitlab.internal.euramax.nl/security/phishing-protection.git

# Install development dependencies
npm install --include=dev

# Start development server
npm run dev

# Run tests
npm test

# Build voor productie
npm run build

Code Standards

  • TypeScript voor alle nieuwe code
  • ESLint configuratie volgens Euramax standards
  • Jest voor unit tests (minimum 80% coverage)
  • Prettier voor code formatting

Git Workflow

# Feature branch naming
feature/JIRA-123-beschrijving

# Commit message format
type(scope): beschrijving

# Bijvoorbeeld:
feat(api): voeg nieuwe threat detection endpoint toe
fix(ui): repareer dashboard loading bug
docs(readme): update installatie instructies

Security Policies

Toegangsbeheer

  • Admin Level: Volledige systeem toegang (IT Security Team)
  • Manager Level: Departement overzichten en rapportages
  • User Level: Basis dashboard en training toegang

Data Retention

Belangrijk: Alle security logs worden 2 jaar bewaard conform Euramax data retention policy.

Incident Response

Critieke bedreigingen:
1. Automatische blokkering binnen 30 seconden
2. Directe notificatie naar IT Security Team
3. Email quarantine van gerelateerde berichten
4. Incident ticket aangemaakt in ServiceNow
